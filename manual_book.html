1. Pendahuluan & Gambaran Umum

BD Project Management adalah aplikasi web manajemen proyek yang dirancang untuk membantu tim dalam merencanakan, melacak, dan mengelola proyek secara efisien. Aplikasi ini dapat diakses melalui browser desktop maupun ponsel.

Fitur Utama

📊 Dashboard

Ringkasan statistik seluruh proyek dan tugas dalam satu halaman.

📋 Kanban Board

Papan visual untuk memindahkan tugas antar status dengan drag & drop.

📅 Timeline & Kalender

Gantt chart dan kalender bulanan untuk melihat jadwal proyek.

👥 Manajemen Tim

Kelola anggota, role, dan permission secara granular.

Teknologi

FrontendReact + Vite + Tailwind CSS v4
Backend / DatabaseSupabase (PostgreSQL)
HostingRailway (Docker)
AutentikasiSupabase Auth (Email/Password)

2. Login & Autentikasi

2.1 Login sebagai Super Admin

Super Admin adalah akun bawaan yang memiliki akses penuh ke semua fitur tanpa perlu didaftarkan di database.

  1. Buka halaman aplikasi di browser.
  2. Pada field Username / Email, ketik: admin
  3. Pada field Password, ketik: admin
  4. Klik tombol Masuk.
💡 Tips Anda juga dapat login admin melalui tombol "Need Help?" di bagian bawah halaman login, yang akan mengaktifkan mode admin khusus.

2.2 Login sebagai User Biasa

  1. Masukkan username (tanpa @bd.com) atau email lengkap.
  2. Masukkan password yang telah diberikan oleh Admin.
  3. Klik Masuk.
ℹ️ Catatan Jika Anda memasukkan username tanpa domain (misal: andi), sistem secara otomatis akan menambahkan @bd.com menjadi andi@bd.com.

2.3 Registrasi (Daftar Akun Baru)

  1. Klik link "Belum punya akun? Daftar" di bawah tombol Masuk.
  2. Isi Nama Lengkap, Username, dan Password.
  3. Klik tombol Daftar.
  4. Akun baru akan berstatus "Menunggu Persetujuan" (pending).
  5. Hubungi Admin untuk mengaktifkan akun Anda.
⚠️ Penting Akun baru tidak bisa langsung login. Admin harus mengubah status akun dari Pending menjadi Active melalui menu Settings → Team Management.

2.4 Logout

Klik ikon Logout (↪) di bagian bawah sidebar (di samping nama pengguna), lalu konfirmasi dengan klik OK.

3. Navigasi & Antarmuka

3.1 Layout Utama (Desktop)

Tampilan desktop terdiri dari tiga bagian utama:

BagianPosisiFungsi
SidebarKiriNavigasi menu, pemilihan project, dan info user
HeaderAtasJudul halaman, dark mode, notifikasi, dan settings
KontenTengahArea kerja utama sesuai menu yang dipilih

3.2 Sidebar

Sidebar berisi:

  • Logo & Nama Aplikasi — di bagian atas.
  • Project Selector — dropdown untuk memilih project aktif atau "Semua Proyek".
  • Menu Navigasi:
    • Dashboard
    • Overview & Info
    • Timeline & Jadwal
    • Kalender
    • Kanban Board
    • Daftar Tugas
    • Tim & Anggota
  • User Info — nama, role, dan tombol logout di bagian bawah.

3.3 Header

Header berisi:

  • 🌙 / ☀️ Dark Mode Toggle — untuk mengubah tema gelap/terang.
  • 🔔 Notifikasi — menampilkan tugas terlambat, tenggat waktu dekat, dan anggota pending.
  • ⚙️ Settings — hanya muncul untuk admin/Project Manager.

3.4 Tombol Aksi Cepat (FAB)

Tombol bulat berwarna ungu (+) di pojok kanan bawah digunakan untuk menambah tugas baru secara cepat dari halaman manapun.

4. Dashboard

Dashboard menampilkan ringkasan statistik dari seluruh proyek dan tugas:

  • Total Proyek — jumlah proyek yang ada.
  • Total Tugas — jumlah seluruh tugas.
  • Tugas Selesai — jumlah tugas berstatus "Selesai".
  • Anggota Tim — jumlah user terdaftar.

Dashboard secara otomatis menampilkan data sesuai dengan project yang dipilih di sidebar. Jika memilih "Semua Proyek", data dari semua proyek akan ditampilkan.

5. Manajemen Project

5.1 Membuat Project Baru

  1. Klik ikon ＋ (PlusCircle) di sebelah Project Selector pada sidebar.
  2. Isi formulir:
    • Nama Project — nama proyek (wajib).
    • Deskripsi — penjelasan singkat proyek.
    • Tanggal Mulai & Selesai — periode proyek.
    • Client — nama klien/pelanggan.
    • Budget — anggaran proyek.
    • Cover Image — URL gambar cover (opsional).
  3. Klik Simpan.
💡 Tips Setiap project baru otomatis dibuatkan Sub-Project bernama "General" agar Anda bisa langsung menambahkan tugas.

5.2 Mengedit Project

  1. Pilih project dari dropdown sidebar.
  2. Buka halaman Overview & Info.
  3. Edit detail project yang diinginkan.
  4. Perubahan akan otomatis tersimpan.

5.3 Menghapus Project

  1. Pilih project yang ingin dihapus dari dropdown sidebar.
  2. Klik ikon 🗑️ (Trash) di sebelah dropdown.
  3. Konfirmasi penghapusan.
⚠️ Peringatan Menghapus project akan menghapus semua sub-project dan tugas di dalamnya secara permanen. Tindakan ini tidak dapat dibatalkan.

6. Sub-Project

Sub-Project digunakan untuk mengelompokkan tugas-tugas dalam satu proyek menjadi fase, modul, atau kategori.

6.1 Membuat Sub-Project

  1. Buka halaman Overview & Info dari project yang dipilih.
  2. Klik tombol Tambah Sub-Project.
  3. Isi nama, tanggal mulai, dan tanggal selesai.
  4. Klik Simpan.

7. Manajemen Tugas (Task)

7.1 Membuat Tugas Baru

  1. Klik tombol ＋ (bulat ungu) di pojok kanan bawah.
  2. Isi formulir tugas:
    • Judul Tugas — nama tugas (wajib).
    • Project & Sub-Project — tugas ini masuk ke mana.
    • Status — Akan Dikerjakan, Sedang Proses, Review, atau Selesai.
    • Prioritas — Tinggi, Sedang, atau Rendah.
    • Ditugaskan Ke — pilih anggota tim.
    • Tanggal Mulai & Deadline.
  3. Klik Simpan Tugas.

7.4 Mengedit Tugas

Klik pada tugas di halaman Kanban, Daftar Tugas, atau Kalender untuk membuka modal edit. Ubah detail yang diinginkan, lalu klik Simpan Tugas.

7.5 Menghapus Tugas

Pada halaman Daftar Tugas, klik ikon 🗑️ di baris tugas yang ingin dihapus, lalu konfirmasi.

8. Overview & Info Project

Halaman ini menampilkan informasi detail dari project yang dipilih:

  • Informasi Project — nama, deskripsi, client, budget, tanggal, dan cover image.
  • Daftar Sub-Project — semua sub-project dalam project ini.
  • Progress Bar — persentase tugas yang telah selesai.
  • Resources — link dan file terkait project (Figma, GitHub, dokumen, dll).
  • Health Status — indikator kesehatan proyek berdasarkan deadline.
ℹ️ Catatan Halaman Overview hanya menampilkan detail project jika Anda telah memilih satu project di sidebar. Jika memilih "Semua Proyek", akan menampilkan ringkasan semua project.

9. Timeline & Jadwal

Halaman Timeline menampilkan Gantt Chart interaktif yang memvisualisasikan jadwal proyek, sub-project, dan tugas dalam format horizontal timeline.

10. Kalender

Kalender menampilkan tugas-tugas dalam format kalender bulanan. Setiap tugas ditampilkan pada tanggal deadline-nya.

Fitur Kalender

  • Navigasi bulan dengan tombol ◀ / ▶.
  • Klik pada tugas di kalender untuk mengedit tugas tersebut.
  • Tugas ditampilkan dengan warna sesuai statusnya.
  • Data difilter sesuai project yang dipilih di sidebar.

11. Kanban Board

Kanban Board menampilkan tugas dalam 4 kolom status yang memungkinkan visualisasi alur kerja:

KolomKeterangan
Akan DikerjakanTugas baru / backlog
Sedang ProsesTugas yang sedang aktif dikerjakan
ReviewTugas selesai, menunggu review

Cara Menggunakan Kanban

  • Ubah status tugas dengan mengklik tombol status pada kartu tugas.
  • Klik kartu tugas untuk mengedit detail.
  • Setiap kartu menampilkan: judul, prioritas, assignee, dan tanggal.

13. Tim & Anggota

Halaman ini menampilkan semua anggota tim beserta role dan statusnya.

Informasi Anggota

  • Avatar — foto profil atau inisial nama.
  • Nama & Email.
  • Role — peran dalam tim (Project Manager, Developer, dll).
  • Status — Active, Pending, atau Inactive.

Aksi Admin

  • Approve/Reject anggota dengan status Pending.
  • Edit profil anggota (nama, role, status, warna).
  • Hapus anggota dari tim.

14. Pengaturan (Settings)

Menu Settings hanya dapat diakses oleh pengguna dengan permission canManageRoles (biasanya Project Manager atau Super Admin). Akses melalui ikon ⚙️ di header.

14.1 Tab: Team Management

Kelola anggota tim:

  • Tambah Anggota Baru — dengan email dan password (langsung aktif, tidak perlu registrasi).
  • Edit Anggota — ubah nama, role, status, dan warna.
  • Hapus Anggota — hapus dari sistem.
  • Ubah Password — hanya bisa untuk diri sendiri.

14.2 Tab: Role Management

Kelola role dan permission:

  • Buat Role Baru — tentukan nama, warna, dan permission.
  • Edit Role — ubah permission akses.

14.3 Tab: Backup

Export data aplikasi dalam format JSON untuk backup.

15. Sistem Role & Permission

Setiap anggota tim memiliki satu Role yang menentukan Permission (hak akses) mereka.

Permission yang Tersedia

PermissionKeterangan
canManageTeamMengelola anggota tim (tambah, edit, hapus, approve)
canManageProjectsMengelola project (buat, edit, hapus project & sub-project)
canManageTasksMengelola tugas (buat, edit)
canManageRolesAkses ke Settings & Role Management
canDeleteTasksMenghapus tugas
canDeleteProjectsMenghapus project

Role Default

RolePermission

17. FAQ & Troubleshooting

Q: Saya tidak bisa login, muncul "Invalid login credentials"

A: Pastikan username dan password benar. Jika login sebagai admin, ketik admin di field Username dan admin di field Password. Jika masih gagal, coba clear cache browser.

Q: Akun saya sudah didaftarkan tapi tidak bisa login

A: Akun baru berstatus "Pending". Minta Admin untuk mengubah status Anda menjadi "Active" melalui Settings → Team Management.

Q: Saya tidak melihat tombol tambah project / edit

A: Tombol-tombol tersebut hanya muncul jika role Anda memiliki permission yang sesuai. Hubungi Admin untuk mengubah role/permission Anda.

Q: Data tidak muncul setelah login

A: Coba refresh halaman (F5). Jika masih kosong, periksa koneksi internet. Jika muncul error di console, hubungi Admin.

Q: Bagaimana cara mengganti password?

A: Buka Settings → Team Management → cari nama Anda → klik Edit → masukkan password baru → Simpan. Catatan: Anda hanya bisa mengubah password sendiri.

Q: Bagaimana cara backup data?

A: Buka Settings → Tab Backup → Klik Export. File JSON akan ter-download.

Q: Aplikasi terasa lambat di HP

A: Gunakan browser Chrome atau Firefox terbaru. Tutup tab browser lain yang tidak diperlukan. Pastikan koneksi internet stabil.
